At RubyConf India 2016, Go-Jek Engineering is running a programming contest to find & reward the sharpest coders in the crowd.

Read on to learn more about the challenge & see if you can hack it.

The Game

  1. You have until 9:00am on Sunday (i.e 24 hours from announcement) to submit your solution.
  2. Your solution must be in Ruby and must be object oriented.
  3. Your solution should be a command line application with a standard project structure expected in Ruby.
  4. Usage of external gems/libs is only permitted for your specs eg. Test::Unit, RSpec, FactoryGirl etc.
  5. Contract for your solution
    • It should reads from a file to obtain the seed, the filename should be passed via command line arguments
      Like -> 
      ~$ ./gameoflife.rb seed.txt`
    • It should run 50 iterations, printing the final iteration to STDOUT.
    • It should implement an infinite grid as per the problem statement. Limited grids will be rejected.
    • Sample seed and output format can be found below - 1 represents live cell and 0 represents dead cell
    • More examples can be found on the Wikipedia article linked to above
  6. Your solution should comply with the input and output file formats below - we will be verifying the output using automated tests, so if you don't stick to the format your solution will not be accepted.
  7. Solution does not need to validate seed input data, assume it is correct.
  8. Solution should be published on github.com
  9. Your solution must work for an infinite grid for any set of seed data. You do not need to handle out-of-memory issues for large seed data or seed data that generate substantial changes in 50 iterations.
  10. Unit tests are mandatory. TDD is a huge plus.
