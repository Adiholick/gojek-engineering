GO-JEK Game Of Life Challenge

  
2016-03-19 09:00:00 +05:30


  
Abstract:

  Implement Conway's Game of Life as a command line solution in Ruby.
  
  
Prize:

  3 iPad minis for the top 3 code submissions.
  
  
Rules of the game:


1. You have until 9:00am on Sunday (i.e 24 hours from announcement) to submit your solution.

2. Your solution must be in Ruby and must be object oriented.

3. Your solution should be a command line application with a standard project structure expected in Ruby.

4. Usage of gems/libs is only for your specs eg. Test::Unit, RSpec, FactoryGirl etc.

5. Contract for your solution 
  
      a. It should reads from a file to obtain the seed, the filename should be passed via command 
      line arguments              
      
          Like -> `~$ ./gameoflife.rb seed.txt`
        
      b. It should run 50 iterations, printing the final iteration to STDOUT. 
      
      c. It should implement an infinite grid as per the problem statement. Limited grids will be 
      rejected.
         
      d. Sample seed and output format can be found below - 1 represents live cell and 0 represents 
      dead cell
         
      e. More examples can be found on the Wikipedia article linked to above

7. Your solution should comply with the input and output file formats below - we will be verifying the output using automated tests, so if you don't stick to the format your solution will not be accepted.

8. Solution does not need to validate seed input data, assume it is correct.

9. Solution should be published on github.com

10. Your solution must work for an infinite grid for any set of seed data. You do not need to handle out-of-memory issues for large seed data or seed data that generate substantial changes in 50 iterations.
  
11. Unit tests are mandatory. TDD is a huge plus.


  
Constraints for the problem:


Your submission must adhere to at least one of the following constraints. The more contstraints that are implemented, the more likely you are to win.

Please mention which constraints you have implemented in the README file in your project root.

1. Your solution uses a maximum of immutable objects. The ideal is 0 mutating references in the entire codebase.

2. Your solution does not use conditionals. Please note that hacking this by using a hash or ternary operator does not count, as one example.

3. Your solution does not use looping constructs.
  
  
How to enter:
